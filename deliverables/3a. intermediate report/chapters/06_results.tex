\section{Preliminary Results}

The preliminary results of this project consist of an exploratory data analysis conducted across all three data sources: the Baumkataster, the Sentinel-2 satellite imagery and the OpensStreetMap (OSM) urban context layers. The goal of the analysis was to asses data quality, learn feature distributions and identify which indicators are likely to be useful for training the U-Net model.

\subsection{Baumkataster Analysis}

This section contains some analysis on the Baumkataster data sets.

\subsubsection{Dataset Overview and Tree Counts}

The cleaned Baumkataster data contain 226,998 trees for Vienna, 217,454 for Paris, 233,505 for Hamburg and 43,367 for Barcelona, as shown in Figure~\ref{fig:tree_counts}. Vienna, Paris, and Hamburg are roughly balanced in size and are suitable as training cities. However, Barcelona lacks size attributes and a geometry column and  therefore it will only be used as point locations for cross-city generalization testing.

The available size attributes differ across cities. Vienna provides height categories, trunk circumference, and crown diameter. Paris provides height and circumference. Hamburg provides circumference and crown diameter.

\subsubsection{Tree Size Distributions}

Figure~\ref{fig:size_histogram} shows the distributions of height, circumference, and crown
diameter across cities. The height histogram reveals a data quality issue specific to
Vienna. Its height column stores ordinal category codes from 1 to 8, representing height
ranges from 0--5\,m up to \textgreater35\,m, rather than continuous metre values.

corresponds to the most frequent category (6--10\,m, midpoint 8.0\,m). The planned fix

\subsubsection{Species Composition}

Species diversity is high across all cities, with 670 unique species in Vienna,
996 in Paris, 354 in Hamburg, and 330 in Barcelona. Across all four cities,
86 species are shared, predominantly from the \textit{Abies} and \textit{Acer} genera.
Species identity will not be used as a direct model input, but the diversity and
climate-specific composition inform expectations about canopy shape and size variation
across cities. Figure~\ref{fig:species} shows the ten most common species in each city.
Vienna is dominated by \textit{Acer platanoides} (Norway maple, 19,244 trees) and
\textit{Aesculus hippocastanum} (horse chestnut, 11,661 trees), both classic central
European street tree species. Paris is dominated by \textit{Platanus x hispanica}
(London plane, 38,113 trees). Hamburg is dominated by \textit{Quercus robur} (English oak, 42,965 trees) and \textit{Tilia x europaea} (European lime, 33,852 trees). Barcelona shows a different composition with subtropical species such as \textit{Tipuana tipu} and \textit{Pinus pinea}, reflecting its Mediterranean climate.

\subsection{Sentinel-2 Analysis}

All 12 city-season combinations were successfully loaded. Band statistics were computed across the seven available channels: B02 (blue), B03 (green), B04 (red), B08 (NIR), NDVI, EVI, and SAVI.\\
\\
NDVI values are highest in summer (August) across all cities, with Hamburg
reaching a mean of 0.42 compared to approximately 0.29 in spring and autumn.
This confirms that summer imagery carries the strongest vegetation signal. The
seasonal contrast between summer and autumn NDVI is particularly valuable for
distinguishing deciduous trees from other permanently green surfaces such as
grass or artificial turf.\\
\\
Another data quality issue was identified with the SAVI indices. EVI values
exceed 1.0 for all cities in summer, which is false as EVI is bounded between $-1$ and $+1$. This is a systematic issue across all cities. The root cause of this is that the index the formulas were applied to the raw pixel values stored in the satellite files, which are large integers, instead of the actual light reflectance values they represent. Converting raw values to reflectance requires dividing by 10,000. NDVI is unaffected here because its formula divides two numbers, so the scale cancels out either way. EVI is more complex and returns incorrect results when the input values are not in the expected range.

\subsection{OSM Analysis}

The OSM layers were loaded for all four cities across four feature types: roads, buildings, landuse, and water. Road networks are dominated by \texttt{footway}, \texttt{service}, and \texttt{residential} classes across all cities. Hamburg's OSM extract is noticeably sparser than the other cities, with 24,184 road features compared to 161,378 for Vienna, which will be investigated before using Hamburg as a training city.\\
\\
Building footprints are present for all cities and will serve as reliable negative
labels during training mask generation, as no trees are expected on rooftop
surfaces. Road classes such as \texttt{footway}, \texttt{path}, \texttt{steps},
and \texttt{track} will not be used as hard negative masks, as these routes
frequently pass through parks alongside trees and would introduce false negatives
into the training labels.
